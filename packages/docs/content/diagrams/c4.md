# C4 Diagram

A [C4 diagram](https://c4model.com/) documents software architecture at four
zoom levels — system context, containers, components, code. kymo's editor
reads the [Mermaid](https://mermaid.js.org/syntax/c4.html) C4 syntax
(`C4Context`, `C4Container`, `C4Component`, `C4Dynamic`, `C4Deployment`),
which mirrors the PlantUML C4 macros.

This page works like a quickstart: as you scroll, the pane on the right shows
the source and the preview for the section you're reading. **Copy** grabs the
source; **▶ Open in editor** loads it into
[editor.kymo.studio](https://editor.kymo.studio) (pick **mermaid** in the
diagram-type dropdown when starting from scratch).

> Mermaid marks its C4 support **experimental** — the syntax may still
> change upstream.

<DqSection id="c4-intro">

A context diagram declares people and systems as macros —
`Person(alias, label, ?description)`, `System(…)`, with `_Ext` variants for
things outside your boundary — and connects them with
`Rel(from, to, label, ?technology)`.

</DqSection>

<DqSection id="c4-boundary">

## Boundaries and containers

One level deeper, `C4Container` zooms into a system:
`Container_Boundary(alias, label) { … }` draws the system's edge, and each
`Container(alias, label, technology, description)` is an application or data
store inside it. The same pattern continues with `C4Component` inside a
container.

</DqSection>

> **Status.** C4 previews on this page and in the editor use the Mermaid
> renderer; importing C4 diagrams into kymo's own pipeline (native
> SVG/PNG/PDF rendering) is on the roadmap.
