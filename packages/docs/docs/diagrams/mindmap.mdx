# Mindmap

A mindmap radiates ideas out from a central root — hierarchy is the whole
syntax. kymo's editor reads the
[Mermaid](https://mermaid.js.org/syntax/mindmap.html) `mindmap` syntax.

This page works like a quickstart: as you scroll, the pane on the right shows
the source and the preview for the section you're reading. **Copy** grabs the
source; **▶ Open in editor** loads it into
[editor.kymo.studio](https://editor.kymo.studio) (pick **mermaid** in the
diagram-type dropdown when starting from scratch).

<DqSection id="mindmap-intro">

**Indentation is structure**: each level of indentation makes a node a child
of the nearest less-indented line above it. There are no operators — just an
outline.

</DqSection>

<DqSection id="mindmap-shapes">

## Node shapes

Wrap a node's text in delimiters to change its shape — the same vocabulary
as flowcharts, plus two of its own:

| Syntax | Shape |
|--------|-------|
| `id[text]` | square |
| `id(text)` | rounded |
| `id((text))` | circle |
| `id))text((` | bang |
| `id)text(` | cloud |
| `id{{text}}` | hexagon |

Without delimiters, the line itself is the label.

</DqSection>
