# State Diagram

A state diagram describes the behaviour of a system: the states it can be in,
and the transitions that move it from one state to another. kymo's editor
reads the [Mermaid](https://mermaid.js.org/syntax/stateDiagram.html)
`stateDiagram-v2` syntax.

This page works like a quickstart: as you scroll, the pane on the right shows
the source and the preview for the section you're reading. **Copy** grabs the
source; **▶ Open in editor** loads it into
[editor.kymo.studio](https://editor.kymo.studio) (pick **mermaid** in the
diagram-type dropdown when starting from scratch).

<DqSection id="state-intro">

The example on the right models a moving object: `[*]` is both the start and
the end state, and each `-->` is a transition.

</DqSection>

<DqSection id="state-states">

## States and transitions

A bare identifier declares a state. Give it a human-readable description with
a colon (`s1 : description`) or the `state "description" as s1` form. A
transition is `from --> to`, with an optional `: label` describing what
triggers it.

`direction LR` (or `TB`, `BT`, `RL`) sets the flow of the layout.

</DqSection>

<DqSection id="state-composite">

## Composite states

A state can contain its own sub-state machine: `state Name { … }`. Composite
states nest, and the inner machine has its own `[*]` start state. Transitions
may connect composite states to outside states — but not a state *inside* one
composite to a state inside another.

</DqSection>

<DqSection id="state-choice">

## Choice, fork and join

`state name <<choice>>` declares a diamond that branches on a condition.
`<<fork>>` splits a transition into parallel paths and `<<join>>` brings them
back together.

</DqSection>

<DqSection id="state-concurrency">

## Concurrency

Inside a composite state, a `--` line splits the body into regions that run
in parallel — the example shows the classic keyboard NumLock/CapsLock pair.

</DqSection>

<DqSection id="state-notes">

## Notes

Attach commentary with `note right of X … end note` (or `left of`), or the
single-line form `note left of X : text`.

</DqSection>
