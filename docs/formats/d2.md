# D2 — Element Mapping (import + export)

This is the **bidirectional** mapping between [D2](https://d2lang.com) and the kymo
flowchart IR (`crate::flowchart`). kymo is a *spoke* on the flowchart hub: it **imports**
the D2 flowchart subset (`crate::d2::parse` → IR → `layout_flowchart` → `.kymo.json` / SVG)
and **emits** D2 from the IR (`flowchart::emit::to_d2`). Cite this document by
`document_id` (`D2-MAP-001`).

> **Upstream source.** D2 has no ISO/OMG specification; the source of record is the D2
> language documentation — <https://d2lang.com>. This is the kymo **subset mapping**, not a
> clause-mirror. D2 is a rich, general language (vars, imports, classes, SQL/UML shapes,
> globs, theming); kymo reads/writes only the **flowchart subset** (nodes, shapes, edges,
> containers, direction). Anything unrecognised is skipped on import, so a richer D2 file
> still imports its graph skeleton.

## 1. Direction

| D2 | kymo `Direction` | screen |
|---|---|---|
| `direction: down` (default) | `Tb` | top→bottom |
| `direction: up` | `Bt` | bottom→top |
| `direction: right` | `Lr` | left→right |
| `direction: left` | `Rl` | right→left |

Top-level only; a `direction` inside a container is ignored.

## 2. Node shapes

`id: "label" { shape: X }` (import) ↔ `id: "label" { shape: X }` (export). Default
rectangle → `box` (no `shape` emitted).

| D2 `shape` | kymo `Shape` | export back as |
|---|---|---|
| (none) / `rectangle` / `square` | `box` | (none) |
| `circle` | `circle` | `circle` |
| `diamond` | `diamond` | `diamond` |
| `hexagon` | `hex` | `hexagon` |
| `cylinder` / `stored_data` / `page` | `cylinder` | `cylinder` |
| `oval` | `badge` (stadium) | `oval` |
| other | `box` (fallback) | — |

A bare `id` (no `:`/block) imports a node whose label is the id. Labels may be quoted
(`"…"`, surrounding quotes stripped) or bare.

## 3. Edges

`a -> b` connectors, optional `: "label"` and a trailing `{ … }` style block.

| D2 | kymo `Edge` |
|---|---|
| `a -> b` | arrow (default) |
| `a -- b` | `no_arrow` |
| `a <-> b` | arrow (bidirectional collapses to one arrow) |
| `a <- b` | arrow, **reversed** (`b → a`) |
| `… : "label"` | `Edge.label` |
| `… { style.stroke-dash: 3 }` | `dashed` |

Chains `a -> b -> c` expand to one edge per segment (the trailing label/dash applies to
the last segment). On **export**, dashed → `{ style.stroke-dash: 3 }`, `no_arrow` → `--`.

## 4. Containers ↔ subgraphs

A D2 container `g: "title" { … }` ↔ a kymo `Subgraph` (cluster region). Member nodes are
declared inside the block; **import resolves qualified edge refs** (`g.a`) to the leaf id
(`a`) and records membership. On **export**, members nest inside `g: "title" { … }` and
edges reference them by the qualified path `g.a`.

## 5. Round-trip & gaps

- **`mmd/dot → d2 → kymo`** round-trips the graph; `to_d2` output re-imports to the same
  IR (ids, labels, shapes, edges, membership). Node *declaration order* is normalised (the
  emitter groups container members) — semantically identical, layout-tiebreak may differ.
- **Not read (skipped):** `vars` / `...@import` / globs; `classes` and `style.*` beyond
  `stroke-dash`; SQL-table / UML / sequence shapes; markdown/LaTeX labels; connection
  chaining glyphs other than `-> / -- / <-> / <-`; nested-container directions.
- **Whitespace:** connectors must be space-flanked (`a -> b`, not `a->b`) for the import to
  recognise an edge — matching what `to_d2` emits and idiomatic D2.
