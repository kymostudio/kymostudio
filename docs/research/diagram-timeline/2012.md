## Highlights

### 1. ArchiMate 2.0 released (and the C4 model takes shape)

The Open Group released **ArchiMate 2.0** on 31 January 2012, aligning the EA modeling language with TOGAF's ADM and adding Motivation and Implementation/Migration extensions. In the same period Simon Brown's **C4 model** (named ~2011) pushed the opposite direction: four nested diagram levels instead of a sprawling formal vocabulary. The tension between formal completeness and pragmatic minimalism is the one every diagram language navigates.

### 2. Diagramly rebrands as draw.io and goes fully JavaScript

JGraph dropped the Java-applet pieces, relaunching the now fully client-side editor at **draw.io** ("the .io suffix is a lot cooler than .ly"). Free, no-signup, mxGraph-powered — the beginning of the ascent that made draw.io/diagrams.net the most-used free diagram tool of the next decade.

### 3. GoJS released by Northwoods Software

Northwoods — maker of the GoDiagram/JGo component family since the 1990s — launched **GoJS**, a canvas-based JS library for building interactive flowcharts, org charts, BPMN editors and visual languages. With mxGraph and JointJS it completed the commercial "diagramming as an embeddable web component" stack that 2010s SaaS products were built on.

## Context

C4's influence grew through the 2010s (Structurizr, C4 support in PlantUML/Mermaid). ArchiMate continued to 3.x. Both reinforced that *which abstraction levels* a notation exposes matters as much as its shapes.
