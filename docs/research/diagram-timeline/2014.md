## Highlights

### 1. Mermaid created — Markdown-style diagrams-as-code

**Mermaid.js** was created in 2014 by Knut Sveidqvist — reportedly after losing a Visio file — rendering flowcharts and sequence diagrams from an intentionally Markdown-like text syntax, entirely client-side in JavaScript. Two design choices made it the most widely adopted diagrams-as-code tool of the next decade: syntax that reads like Markdown (no Java toolchain), and rendering that embeds anywhere a `<script>` runs — which later let GitHub, GitLab, Notion, and Obsidian render it natively. See `RES-MERMAID-D2-001`.

### 2. bpmn.io announced by Camunda

Announced 25 February 2014: an open-source, embeddable **BPMN 2.0 rendering toolkit and web modeler** ("BPMN everywhere, for everyone"), with full render support by May. It did for process diagrams what mxGraph did for general diagrams — made standards-compliant BPMN viewing/editing a drop-in web component, now embedded in hundreds of products.

### 3. nomnoml released

Daniel Kallin published **nomnoml** (repo created June 2014), "the sassy UML diagram renderer" — a minimal text-to-UML language rendered client-side with deliberately friendly styling. Alongside Mermaid, it marks 2014 as the year diagram-as-code went from server-side niche (PlantUML) to a browser-native movement.

## Context

Mermaid closes Era B: the text-to-diagram thread that started with Graphviz's DOT now had a web-native, doc-native champion. Its 2022 GitHub integration and 85k+ GitHub stars came from this 2014 seed. (Structurizr — Simon Brown's C4 tooling — was also taking shape around 2014–2015, though its exact launch date is unpinned.)
