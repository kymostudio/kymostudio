## Highlights

### 1. Microsoft acquires Visio; Graphviz open-sourced

On **January 7, 2000** Microsoft completed its ~$1.375B acquisition of **Visio Corporation**, folding the leading business-diagramming application into Office. The same year, **AT&T released Graphviz as open source**, turning its DOT language and automatic layout engines (`dot`, `neato`) into the de-facto open toolkit for *programmatic* graph drawing. These are the two poles of diagramming for the next decade: manual WYSIWYG desktop authoring (Visio) versus declarative, code-driven layout (Graphviz) — the second pole being the direct ancestor of every diagram-as-code tool.

### 2. Sparx Systems releases Enterprise Architect 1.0

Released August 2000: a full UML modeling tool with code reverse-engineering **at a fraction of Rational Rose's four-figure pricing**. EA's price disruption — "UML for every developer, not every project lead" — made it one of the most-installed modeling tools of the following decade.

### 3. GraphML initiated at Graph Drawing 2000; yWorks founded

The GD Steering Committee convened a file-format workshop at GD 2000 (Williamsburg), spawning the task group that produced **GraphML** — the XML graph-exchange format still used by yEd, Gephi, and NetworkX. **yWorks GmbH** (yFiles layout library, later yEd) was founded the same year as a University of Tübingen spin-off, commercializing automatic layout.

## Context

Visio's dominance set the bar that web tools (Gliffy, Lucidchart, draw.io) would later try to clear in the browser. Graphviz quietly became infrastructure — embedded in Doxygen, profilers, and countless pipelines — proving automatic layout was good enough to ship. Graphviz's DOT is arguably the first widely used diagram *language*.
