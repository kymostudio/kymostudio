## Highlights (projected)

### 1. Agent-design platforms emerge as a category

A new kind of surface enters the list: not a whiteboard with AI bolted on, but a design environment built for mixed human-agent teams — agents propose system designs as structured models, humans annotate and approve, and every element stays bound to the artifact it describes. The category is to FigJam what Figma was to Photoshop: same canvas, different assumption about who is drawing.

### 2. Review replaces authoring as the human's job

In agent-heavy organizations the modal human interaction with a diagram is no longer drawing it but approving it. Diagram surfaces grow review primitives — proposals, diffs, sign-offs — and the "suggested edit" becomes as load-bearing for diagrams as it was for documents.

### 3. Rendering becomes a service layer

With assistants and platforms all emitting structured models, deterministic rendering — same model, same pixels, any backend — consolidates into shared infrastructure, the way Kroki prefigured in 2019. Layout quality stops being a product moat and becomes a commodity the way font rasterization did.

## Context

Eraser's exit marks the end of the "AI co-pilot for diagrams" generation — not because it failed, but because its feature set became the baseline everywhere. Categories die by absorption more often than by defeat.

## Signals

- Eraser's AI-copilot positioning (2024–2025) — the feature set being absorbed
- Kroki as the render-gateway prototype — <https://kroki.io>
- Review-workflow primitives in code hosting (the pattern being borrowed) — GitHub PR model
