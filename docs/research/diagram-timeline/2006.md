## Highlights

### 1. BPMN becomes an OMG standard

The OMG released the formal **BPMN specification in February 2006**, following the BPMI merger. OMG governance turned BPMN from a promising notation into a globally adopted standard — the same trajectory UML took in 1997 — bringing cross-vendor interoperability and the credibility enterprises needed to standardize on it.

### 2. JGraph releases mxGraph — the first JavaScript in-browser diagramming library

JGraph Ltd publicly released **mxGraph**, a fully client-side JS diagramming library rendering via SVG/VML — no Flash, no Java applet. It proved a full diagram editor could run in a plain web page, and it is the engine that later powered Diagramly/draw.io: arguably the most consequential piece of diagramming infrastructure of the era.

### 3. Eclipse Graphical Modeling Framework (GMF) 1.0 ships in Callisto

Released 30 June 2006 in Eclipse's first simultaneous release train, GMF generated graphical diagram editors from EMF models. It became the standard substrate for a generation of open-source modeling tools (UML2 Tools, BPMN editors, later Archi), moving diagram-editor construction from hand-coding to model-driven generation.

## Context

BPMN now sat alongside UML under one roof at the OMG, which later enabled shared infrastructure — the Diagram Interchange (DI) metamodel for serializing diagram geometry across both notations. That DI geometry is exactly what kymo's BPMN importer consumes.
