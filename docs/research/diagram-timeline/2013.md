## Highlights

### 1. Browser diagramming lands inside the cloud office

A consolidation year: draw.io rode the Google Drive third-party-app ecosystem to become a default free diagram editor *inside* Drive, while Lucidchart shipped as a web app hosted in **Microsoft Word 2013** with Visio (`.vsd`/`.vsdx`) import. Diagrams stopped being a separate destination and became a feature inside the cloud office suite — completing the move Gliffy started in 2005.

### 2. Visio 2013 ships the new .vsdx file format

Visio 2013 replaced the 20-year-old binary `.vsd` with **`.vsdx`** — an Open-Packaging-Conventions ZIP+XML format — enabling co-authoring, programmatic manipulation without the Visio runtime, and browser rendering via SharePoint. `.vsdx` became the lingua-franca import/export target for every competitor (Lucidchart, draw.io).

### 3. Camunda forks Activiti and launches camunda BPM

On 18 March 2013 Berlin-based Camunda — an Activiti contributor since day one — forked the open-source engine. The fork reset the open-source BPM landscape, produced the decade's leading developer-centric BPMN platform, and directly led to bpmn.io a year later. (Also 2013: bramp's js-sequence-diagrams brought WebSequenceDiagrams-style text-to-sequence rendering into the browser — a direct stylistic ancestor of Mermaid.)

## Context

No single crisp landmark defines 2013 — the durable trend is integration: diagramming as a guest inside Google's and Microsoft's platforms, which set up the deep embedding (GitHub-native Mermaid, FigJam) of the years to follow.
