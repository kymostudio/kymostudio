## Highlights

### 1. First W3C Working Draft of SVG

The W3C published the **first public Working Draft of Scalable Vector Graphics** in 1999 (announcement in February; dated drafts from July and December). SVG defined a vendor-neutral, XML-based, resolution-independent vector format for the web, backed by Adobe, Apple, IBM, Microsoft, Sun, and Visio. For diagramming it is foundational: SVG became the dominant interchange and display format for diagrams online.

### 2. Visio 2000 ships as Microsoft announces the $1.3B acquisition

**Visio 2000** — the last independent release, whose Enterprise edition did network auto-discovery — arrived as Microsoft announced (15 September 1999) it would acquire "the leading supplier of enterprise-wide business diagramming and technical drawing software." The deal validated diagramming as an office-suite-scale category (it closed in January 2000).

### 3. XMI 1.0 and UML 1.3: model interchange standardizes

OMG's **XML Metadata Interchange 1.0** (February 1999), backed by 29+ vendors, gave UML models a vendor-neutral serialization for the first time, while **UML 1.3** (June 1999) was the first major cleanup revision. "A diagram is data you can exchange between tools" became an official standards position — a thread that runs to GraphML, BPMN DI, and beyond.

## Context

It took two more years for SVG to reach Recommendation status (2001), and browser support lagged for most of the 2000s — partly why desktop tools still dominated authoring through this era. But the substrate for the web-diagramming age was now being laid down.
