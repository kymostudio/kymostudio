# editor bench — 2026-06-12 day summary: four rounds, share links to green

One day, four benchmarking rounds, fourteen PRs. This rolls up the per-round
snapshots ([archive](README.md)) and articles ([research](../research/README.md))
into one view: where the editor's first load started, what moved it, and what
each move cost.

## Where it started

The day opened with a user-reported bug, not a benchmark: a mermaid share link
rendering as **empty shapes** — the SVG sanitizer stripped `<foreignObject>`,
deleting every label while the status bar stayed green (PR #279). Profiling
that same link found the second problem: on cold Fast 4G the diagram appeared
at **~4.3 s**, with ~60 % of all bytes spent on a wasm engine the page never
used and the kroki render request leaving ~1.9 s in (PR #282). The bench was
built that morning *because* both failures were silent — one needed assertions
on content, the other on traffic.

## Architecture after the day

| Kind | Renderer | First-paint path | Wire cost |
|---|---|---|---|
| kymo | lean wasm (`--features wasm`), separate `.wasm`, streaming compile | local | 290 KB engine (was 2,465) |
| mermaid | mermaid.js in-browser (one prebundled chunk) | share links race the edge cache 900 ms — a hit costs 0 mermaid bytes | 760 KB brotli, only on miss/edit; cached immutable |
| 27 kroki kinds | kroki.io via `mcp.kymo.studio/api/render` edge cache | early kick-off from inline HTML script | SVG only; direct-kroki fallback |

npm / vscode artifacts are untouched throughout: they build `wasm,pdf` and
keep the full export surface.

## Negative results (measured, kept)

1. **`wasm-opt -Oz` worsens the wire**: −21 % raw but **+70 KB brotli** —
   stayed out of CI. Grade size work in transfer bytes, not file bytes.
2. **UMD `mermaid.min.js` breaks under bundling** — script-tag-scoped
   (top-level `var` + `globalThis` chain).
3. **Localhost cannot see request waterfalls**: naive `import("mermaid")` was
   989 ms locally and **5,571–14,257 ms** on throttled prod (~25 nested
   chunks × 165 ms RTT) — caught by the new nonce-per-load `mermaid-fresh`
   scenario *before* the snapshot landed, fixed by pre-bundling (#307).

## Standing items

- The 900 ms race window is dead time on the fresh-mermaid path — speculative
  prefetch would trade hit-case bytes for miss-case latency; measure first.
- icons-manifest JSON (~48 KB gz) still rides the engine chunk.
- The bench measures two share scenarios and a sample on one desktop: nothing
  covers second-edit latency, collaborative rooms, or mobile CPUs. Extending
  the bench ranks above further optimization.
